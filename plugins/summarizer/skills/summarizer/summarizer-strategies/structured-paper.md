# Strategy: Structured Paper

Suitable for: transcripts, meeting notes, lecture recordings, interview notes, unstructured source material that needs to be turned into a readable document.

## Output file naming convention
Derive output filenames from the input filename (without extension):
- Paper output: `<input_filename_without_ext> - Paper.md`

Example: If the input file is `meeting-notes.md`, then:
- Paper output → `meeting-notes - Paper.md`

## Task

Transform the input text (e.g. from a transcript, notes, or other sources) into a well-readable, structured, factual paper. Retain all content details, shorten as little as possible.

The structure of the paper should be as follows:
- Concise headline
- Abstract/summary of the most important aspects
- Paper, divided into approx. 3–5 content chapters or sections

Please note the following guidelines when formulating the text:
Change from a personal to a neutral narrative perspective.
Write in a neutral and factual style
Otherwise, do not remove anything, do not shorten, and delete as little information as possible
Add chapter and subheadings (and include the appropriate timestamp in parentheses)

Format using Markdown.
